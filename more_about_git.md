# More

## 一、Fork 和 Clone 的区别
Fork 是在 GitHub 等网站平台上 复制一个远程仓库到你的账户 ，这是 服务器端操作 ，会在你的 GitHub 账户下创建一个完全独立的仓库副本，归你所有，你可以自由修改而不影响原仓库。 Clone 是将远程仓库 下载到你的本地电脑 ，这是 本地操作 ，会在你的电脑上创建一个工作目录，让你可以在本地编辑代码、提交更改，Clone 的可以是自己的仓库、Fork 后的仓库，或任何公开仓库。

## 二、参与项目合作提 PR 的标准流程
第一步，在 GitHub 上 Fork 原项目到你的账户；第二步， Clone 你的 Fork 仓库到本地进行开发；第三步，在本地创建功能分支、修改代码并提交；第四步，将修改 Push 到你的 Fork 仓库；第五步，在 GitHub 上从你的 Fork 仓库向 原仓库 创建 Pull Request ，请求原作者审核并合并你的代码。

### 场景 1：没有写入权限（用 Fork） 
当你想修改某个开源项目但没有 push 权限时，需要先 Fork 到自己的账户，开发完成后 Push 到自己的 Fork，然后提 PR 请求原作者合并。这种方式适用于：开源项目贡献、陌生人的仓库、你没有被添加为协作者的情况。

### 场景 2：有写入权限（直接 Clone）
 当你参与团队项目或自己的项目且有 push 权限时，可以直接 Clone 仓库，开发完成后直接 Push 到原仓库（或提 PR 给队友审核）。这种方式适用于：你自己的仓库、你是协作者/团队成员、组织内部项目（有权限）的情况。
